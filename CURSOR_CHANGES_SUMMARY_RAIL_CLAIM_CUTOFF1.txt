CODE REVIEW FIXES APPLIED
-------------------------

OBSERVATION 1: SELECT queries inside LOOP AT lt_pod
  FIX: All cutoff-related SELECTs moved before LOOP AT lt_pod.
       Loop uses READ TABLE ... BINARY SEARCH on pre-filled internal tables.

OBSERVATION 2-4: Unknown fields LT_RIL_CUTOFF, LW_RIL_CUTOFF, LW_VTTK_RIL
  ROOT CAUSE: Local TYPES were declared after the first DATA statement.
               In ABAP function modules, all TYPES must precede all DATA.
  FIX:
    - All TYPES (including lty_ril_cutoff, lty_vttk_shp, etc.) moved above
      the first DATA statement.
    - Replaced single structure lw_vttk_ril with table type lty_vttk_shp /
      work area lw_vttk_shp (clear naming, valid declaration order).

--------------------------------------------------------------------------------
LOGIC: BULK SELECT (before LOOP AT lt_pod)
--------------------------------------------------------------------------------

  1. SELECT lt_ril_cutoff FROM zlog_exec_var (ZSCM_RIL_CUTOFF_DATE, ACTIVE = X)

  2. Build lt_pod_tk from lt_pod (COLLECT unique tknum)

  3. IF lt_ril_cutoff and lt_pod_tk not initial:
       SELECT lt_vttk_shp FROM vttk FOR ALL ENTRIES lt_pod_tk

       IF i_srchtyp = gc_2:
         SELECT lt_rail_lr FROM ztrstlmnt_rail FOR ALL ENTRIES lt_pod_tk

       IF i_srchtyp = gc_1:
         SELECT lt_ytts2_shp FROM yttstx0002 FOR ALL ENTRIES (shnumber)
         SELECT lt_ytts1_shp FROM yttstx0001 FOR ALL ENTRIES (shnumber)
         SELECT lt_ytts2_rep FROM yttstx0002 FOR ALL ENTRIES lt_ytts1_shp
                (report_no fallback)

  4. SORT all internal tables by key field


--------------------------------------------------------------------------------
LOGIC: LOOP AT lt_pod (READ TABLE only)
--------------------------------------------------------------------------------

  READ TABLE lt_vttk_shp BY tknum
  Resolve lw_rail_lr_dt from lt_rail_lr OR lt_ytts2_shp / lt_ytts1_shp + lt_ytts2_rep
  READ TABLE lt_ril_cutoff BY shtyp + transplpt
  Compare LR_DT vs cutoff REMARKS -> Fr8First error if greater
